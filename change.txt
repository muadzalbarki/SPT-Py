1. QPAINTER FIX — AnimationManager._remove_effect
---------------------------------------------------
Problem:  Every fade_in/fade_out call created a new QGraphicsOpacityEffect and
          set it on the widget via setGraphicsEffect() without removing the old
          one. Qt then got two painters on the same paint device, causing:
          - QPainter::begin: A paint device can only be painted by one painter
          - QPainter::translate: Painter not active
          - QPainter::restore: Unbalanced save/restore
          - QWidgetEffectSourcePrivate::pixmap: Painter not active
          The old effect also leaked memory.

Fix:      Added AnimationManager._remove_effect(widget) which:
          1. Calls widget.graphicsEffect() to get the old effect
          2. Calls widget.setGraphicsEffect(None) to detach it
          3. Calls old.deleteLater() to free it
          Called at the top of both fade_in() and fade_out()

File:     app/core/animation_manager.py

---

2. PAGE SWITCH FIX — clean transition + forced repaint
--------------------------------------------------------
Problem:  When switching pages via sidebar, the old widget remained partially
          visible (stale render), the new widget didn't paint until the user
          moved the mouse, and animations didn't play.

Fix:      In MainWindow._switch_page():
          1. AnimationManager._remove_effect(old) → clean up old opacity
          2. old.hide() → ensure old page is fully hidden
          3. pages.setCurrentIndex(index) → switch
          4. current.show() + current.update() + current.repaint()
          5. QApplication.processEvents() → flush the paint queue
          6. Then refresh + fade_in on the new page

File:     app/ui/main_window.py

4. PARTICIPANT FORMATTER — block-based template generation
------------------------------------------------------------
Problem:  The old code had a hardcoded hack in document_service.py that
          auto-filled {Nama} and {Jabatan} with the ketua komisi's data,
          ignoring selected participants. Multi-participant output was broken:
          - "MEMERINTAHKAN" section showed only the ketua
          - "DAFTAR PESERTA PENDALAMAN MATERI" showed all ketua names
          - No looping/block structure for multiple participants

Fix:      Created participant_formatter.py with three exported functions:
          - format_peserta_block(pegawai_list) → pre-formatted numbered blocks
            each containing Nama, Pangkat/gol, NIP, Jabatan per participant
          - format_daftar_peserta_block(pegawai_list) → compact numbered list
            of "Nama — Jabatan" per participant
          - build_participant_context(pegawai_list) → returns dict with:
              peserta_terpilih: list[dict] (for experimental XML loops)
              peserta_block: str (pre-formatted, use in template as
                             {peserta_block})
              daftar_peserta_block: str (use as {daftar_peserta_block})
              jumlah: str (count)
              Nama, NIP, Jabatan, Pangkat/gol: first participant
              (fallback, NOT overriding block system)

======================================================

1. PARTICIPANT FORMATTER — lowercase keys
------------------------------------------
Problem:  Template SPT Setwan.docx had participant table with {nama}, {nip},
          {jabatan}, {pangkat/gol} (LOWERCASE). Our build_participant_context()
          only provided uppercase keys {Nama}, {NIP}, {Jabatan}, {Pangkat/gol}.
          TemplateEngine does literal string replacement — case mismatch means
          nothing got replaced. The output showed blank/unchanged placeholders.

Fix:      Added lowercase keys alongside uppercase:
            nama, nip, jabatan, pangkat/gol  → for template table rows
            Nama, NIP, Jabatan, Pangkat/gol  → for daftar peserta section

v3.1 — LINE BREAK FIX + NAMA KETUA SAJA
=========================================
Date: 2026-06-05

1. LINE BREAK — newline → <w:br/> di XML Word
-------------------------------------------------
Problem:  Template engine mengganti {placeholder} dengan string multi-line
          (\n), tapi Word hanya menampilkan newline sebagai spasi karena
          XML word/document.xml menyimpan teks dalam elemen <w:t>. Newline
          di dalam <w:t> tidak menghasilkan line break — Word memerlukan
          elemen <w:br/> untuk baris baru.

Fix:      Di template_engine.py, setelah semua placeholder direplace dalam
          teks murni (bukan XML), post-process dilakukan pada tree XML:
          - Parse teks hasil replace menjadi tree ElementTree
          - Cari semua elemen <w:t> yang mengandung \n
          - Split setiap <w:t> by \n
          - Untuk setiap bagian split, sisipkan <w:br/> di antaranya
          - Hapus <w:t> asli, ganti dengan fragment baru
          Hasil: setiap \n dalam teks menjadi <w:br/> di XML Word.

File:     app/services/template_engine.py

---

2. NAMA KETUA SAJA — tanpa jabatan
-------------------------------------
Problem:  Context key {nama_dan_jabatan_ketua_{komisi}} menghasilkan
          "Nama\nJabatan" (2 baris). Tapi template surat hanya perlu nama
          ketua saja — jabatan sudah ditampilkan terpisah di header/tabel.

Fix:
          a. Ubah key: {nama_dan_jabatan_ketua_{komisi}}
             → {nama_ketua_{komisi}}
          b. get_ketua_komisi() kembali hanya nama (str), bukan
             "Nama\nJabatan"
          c. document_service.py context key: 'nama_dan_jabatan_ketua_{k}'
             → 'nama_ketua_{k}'
          d. generate_page.py preview: nama_dan_jabatan_ketua → nama_ketua
          e. Edit SPT Setwan.docx XML untuk rename placeholder

Files:

3. PESERTA BLOCK — format rapi untuk Word
--------------------------------------------
Problem:  Format peserta_block sebelumnya tidak konsisten — ada extra
          baris kosong di antara blok peserta, dan daftar_peserta_block
          tidak memiliki jarak antar entry.

Fix:      Format baru peserta_block:
            {i}.
            (blank)
            Nama\t\t: {nama}
            (blank)
            Pangkat/gol\t\t: {pangkat}
            (blank)
            NIP\t\t: {nip}
            (blank)
            Jabatan\t\t: {jabatan}
            (blank) — antara blok (bukan 2 blank)
          Format daftar_peserta_block:
            {i}.{nama} — {jabatan}
            (blank) — antar entry
          "".join(blocks) bukan "\n\n".join(blocks) agar tidak
          menghasilkan blank berlebih. Trailing \n setelah {jabatan}
          dikendalikan manual lewat f-string.
