PROJECT: SPT-DPRD (Sistem Otomatisasi Surat Pemerintahan)
=========================================================
Aplikasi desktop berbasis PySide6 untuk Sekretariat DPRD Kota Salatiga.
Bertujuan mengotomatiskan pembuatan surat tugas (SPT), surat permohonan,
dan surat kunjungan kerja menggunakan template .docx dengan placeholder
{ } yang diisi secara dinamis. Database menggunakan SQLite + SQLAlchemy
dengan fitur CRUD pegawai, generate surat, ekspor PDF (via LibreOffice),
serta import/export Excel.

=========================================================
ALUR PENGGUNAAN APLIKASI
=========================================================
1. Kelola data pegawai di menu PEGAWAI (tambah/edit/import/export)
2. Generate surat di menu GENERATE (wizard 4 langkah, pilih peserta,
   isi form surat, detail kunjungan, preview & generate)
3. Lihat hasil surat di menu RIWAYAT SURAT
4. Ekspor surat ke PDF jika diperlukan

=========================================================
PENJELASAN SETIAP MENU
=========================================================

1. DASHBOARD (Beranda)
   - Halaman utama dengan 2 kartu statistik:
     Total Pegawai, Riwayat Surat
   - Tabel 10 surat terbaru (No Surat, Tanggal, Template, Peserta)

2. PEGAWAI (Data Pegawai)
   - CRUD data pegawai DPRD: Nama, NIP, Jabatan, Pangkat/Gol,
     Komisi, No. HP
   - Pencarian real-time (nama/NIP/jabatan)
   - Import data dari Excel (.xlsx/.xls)
   - Export data ke Excel
   - Edit data via double-click pada tabel
   - Dialog modal untuk tambah/edit pegawai

3. GENERATE (Generate Surat)
   - Inti dari aplikasi - pembuatan surat otomatis
   - Wizard 4 langkah dengan QStackedWidget:
     Step 1 (Pilih Peserta):
       a. Pilih peserta via Participant Checklist (group per komisi:
          Pimpinan, A, B, C dengan collapsible section)
       b. Search bar filter nama/jabatan
       c. Tombol "Pilih Semua" per komisi
     Step 2 (Surat Tugas):
       a. Pilih Komisi (A/B/C)
       b. Nomor Surat (halaman 1) + Nomor SPT (halaman 2-4)
       c. Tanggal Surat (date picker)
       d. Hari/Tanggal Kepergian (text bebas)
       e. Dasar dilakukannya perjalanan dinas
       f. Tujuan dilakukannya perjalanan dinas
     Step 3 (Detail Kunjungan):
       a. Kab/Kota, Provinsi, Tentang, Materi
       b. Hari, Tanggal, Pukul kunjungan
       c. Rincian Jumlah (auto-fill dari peserta + pendamping)
       d. Pendamping: input Nama + Jabatan, tombol Tambah, daftar dinamis
     Step 4 (Signature & Generate):
       a. Preview otomatis: Komisi, Nomor Surat, Tanggal, Dasar,
          Untuk, Jumlah Peserta, Nama Ketua Komisi (auto dari DB),
          Daftar Peserta
       b. Klik "Generate Surat" - sistem mengganti semua placeholder
          dengan data dari form + data pegawai terpilih
   - Navigasi: tombol « Sebelumnya dan Selanjutnya »
   - Step indicator dengan visual active (gold) dan done (green)
   - Progress bar selama proses generate
   - Hasil: file .docx tersimpan + record di database

4. RIWAYAT SURAT (Riwayat Surat)
   - Daftar semua surat yang telah digenerate (max 100 record)
   - Kolom: No Surat, Tanggal, Template, Peserta, File (status)
   - Tombol Refresh untuk memperbarui daftar
   - Tombol Export PDF (memerlukan LibreOffice terinstal)
   - Sorting descending berdasarkan tanggal pembuatan

5. PENGATURAN (Settings)
   - Tentang Aplikasi: versi (v3.7), deskripsi, organisasi
     (Sekretariat DPRD Kota Salatiga)
   - Developer: info peserta magang Prodi TI Fakultas Dakwah UIN Salatiga

=========================================================
TEKNOLOGI
=========================================================
- Python 3.12+ dengan PySide6 (Qt for Python)
- SQLite via SQLAlchemy ORM
- Template docx: placeholder { } replacement via python-docx + lxml
- Ikon: Material Design Icons via qtawesome (mdi.* prefix)
- Font: Inter (sans-serif modern)
- Dark Mode Only: Government Navy + Gold theme via ColorTokens
- Export PDF: LibreOffice headless
- Export/Import Excel: pandas + openpyxl
